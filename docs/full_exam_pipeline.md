
# 🗺️ TOEIC Full Exam Data Pipeline Strategy

"Part 1 (사진 묘사)"은 시작일 뿐입니다.
똑같은 **Gemini Vision** 기술로 **TOEIC LC & RC 전 파트**를 처리할 수 있습니다.
오히려 사진보다 **텍스트(Part 3~7) 인식이 훨씬 정확하고 빠릅니다.**

---

## 🏗️ 파트별 공략법 (Universal Strategy)

우리가 만든 스크립트(`process-gemini.ts`)를 조금만 업그레이드하면 모든 파트를 처리할 수 있습니다.
AI에게 **"이게 몇 파트인지 보고, 그에 맞춰 무조건 JSON으로 뽑아줘"**라고 시키면 됩니다.

| 파트 | 유형 | 특징 | 처리 전략 (Gemini Vision) | 난이도 |
| :--- | :--- | :--- | :--- | :--- |
| **Part 1** | 사진 묘사 | 이미지 인식 필수 | **완료.** (사진 → 4지선다 JSON) | ⭐⭐⭐ |
| **Part 2** | 질의 응답 | **문제지에 아무것도 없음** | ⚠️ **전략 변경 필요**<br>문제지 이미지가 무의미함. (Mark your answer만 써있음)<br>👉 **"스크립트북(해설집)"**을 찍어서 올리거나, 정답만 따로 입력해야 함. | ⭐ |
| **Part 3** | 대화문 | 3문제 세트 + 도표/그래프 | **가능.** (문제 3개 묶음 인식)<br>도표/그래프도 이미지로 인식해서 텍스트로 변환 가능. | ⭐⭐ |
| **Part 4** | 설명문 | 3문제 세트 + 도표/그래프 | **Part 3와 동일.** | ⭐⭐ |
| **Part 5** | 단문 빈칸 | 100% 텍스트 문법 문제 | **매우 쉬움.**<br>사진보다 텍스트 인식이 훨씬 빠르고 정확함. | ⭐ |
| **Part 6** | 장문 빈칸 | 지문 안에 빈칸 4개 | **가능.**<br>지문 문단과 구멍 뚫린 위치를 통째로 JSON화. | ⭐⭐ |
| **Part 7** | 독해 | 이메일, 문자, 기사 등 복잡한 레이아웃 | **Gemini의 장기.**<br>복잡한 레이아웃(표, 신문기사)을 일반 OCR보다 훨씬 잘 이해함. | ⭐⭐⭐ |

## 🚀 업그레이드 계획 (Next Steps)

현재 `process-part1-gemini.ts`를 **`process-universal-gemini.ts`**로 확장합니다.

### 1단계: 프롬프트 통합 (All-in-One Prompt)
AI에게 이미지를 던져주고 이렇게 명령합니다.
> "이 이미지가 Part 1, 3, 4, 5, 6, 7 중 어디에 해당하는지 먼저 판단해.
> 만약 **Part 5**라면 문법 문제 포맷으로, **Part 7**이라면 독해 포맷으로 JSON을 뽑아줘."
*(이렇게 하면 사용자는 일일이 파트를 구분할 필요 없이 폴더째로 돌리면 됩니다.)*

### 2단계: Part 2 (반전) 처리
Part 2는 문제지(Question Book)가 아니라 **스크립트북(Answer Script)**을 스캔해야 학습이 가능합니다.
- **입력:** 스크립트북 사진 (질문과 대화가 적힌 종이)
- **출력:** Part 2 포맷의 JSON (질문 텍스트 + 정답)
